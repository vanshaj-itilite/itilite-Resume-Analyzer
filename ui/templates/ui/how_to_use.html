Guide

How to Use the ITILITE Resume Analyzer

This page walks you through the full workflow: from uploading resumes (single or batch), to AI processing behind the scenes, to exploring the analytics dashboard.

Contents

  • 1. Upload resumes (single & batch)
  • 2. Processing pipeline
  • 3. AI agents behind the scenes
  • 4. Reading the analytics page

Step 1

Upload Resumes (Single or Batch)

Start on the Home / Upload page. Here you can upload either a single resume or a batch of resumes for bulk analysis.

Single upload

  1. Click on the file picker and select a PDF or DOCX resume from your system.
  2. Optionally give this run a descriptive name (e.g. “Senior Backend – Jan batch”).
  3. Choose the AI model / analysis configuration if the option is shown.
  4. Click Analyze to start processing.

Batch upload

  1. Compress all your resumes into a single .zip file.
  2. Upload the zip via the same file picker.
  3. Confirm the total number of resumes detected in the batch summary.
  4. Click Analyze to launch a bulk analysis run.

Supported formats and limits (file types, size, and count per run) are enforced at upload time. If something is not accepted, you’ll see a clear validation message on the page.

Step 2

Processing Pipeline

Once you submit an upload, the system pushes your files into an asynchronous processing pipeline. You can safely navigate away while the analysis runs.

  1. File intake – documents are validated, stored, and assigned an internal run ID.
  2. Text extraction – PDFs and DOCX files are converted into clean, structured text.
  3. Field parsing – basic fields like name, contact details, education, and experience blocks are extracted.
  4. AI scoring – specialized agents (see below) evaluate each candidate on different dimensions.
  5. Aggregation – scores and metadata are pushed into the analytics store used by the dashboard.

You can reopen the same analysis run later from the history or analytics view; there is no need to re-upload files unless the underlying resumes have changed.

Under the hood

AI Agents Behind the Scenes

The analyzer uses multiple focused AI agents instead of a single monolithic model. Each agent is optimized for a specific part of the evaluation, which makes the results more interpretable and controllable.

Technical Agent

Evaluates the candidate’s technical skills based on role, stack, and seniority. It looks at languages, frameworks, systems design exposure, and depth vs. breadth.

Psychometric / Behavioral Agent

Infers behavioral indicators like ownership, collaboration, leadership, and growth mindset from how the candidate describes their work and impact.

Semantic Matching Agent

Compares the resume against the job description or role definition using embeddings, surfacing how well the candidate aligns with the requirements and responsibilities.

Document & Quality Agent

Checks formatting quality, clarity, signal‑to‑noise ratio, and basic hygiene (typos, structure, redundancy), which can influence how easy the resume is to review.

Depending on your configuration, you may see only a subset of these agents. The analytics page shows which dimensions are active for a given run.

Step 3

Reading the Analytics Page

After processing completes, open the Analytics page for the corresponding run. This is where you explore scores, comparisons, and detailed breakdowns for each candidate.

Key areas on the analytics page

  • Run summary – number of resumes processed, time taken, and basic distribution of scores.
  • Candidate table – sortable list of candidates with overall scores and key tags.
  • Dimension breakdown – technical, behavioral, and semantic‑fit scores side by side.
  • Drill‑down view – per‑candidate view with agent‑level explanations and comments.

Use the filters and sorting controls to shortlist the most promising candidates:

  • Sort by overall fit or a specific dimension (e.g., technical depth first).
  • Filter by minimum score thresholds to focus on top percentiles.
  • Open an individual candidate to understand why they scored the way they did.

A good workflow is: first shortlist candidates using the analytics view, then open their original resumes only for the final validation pass. This keeps the review process fast but still human‑in‑the‑loop.
