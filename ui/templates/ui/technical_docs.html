System Architecture and API Reference

Core components, data flow, and operational behavior.

Architecture Flow

Django UI submits analysis requests to FastAPI. Async processing runs through Celery with Redis broker/backend and progress tracking.

Django UI
->
FastAPI
->
Celery
->
Redis
->
Reports

Endpoints

  • POST /analysis - single analysis
  • POST /analysis/async - async file analysis
  • GET /analysis/status/{task_id} - task progress and result

Analysis Dimensions

  • Semantic fit and role relevance
  • Technical depth and skill signal
  • Behavioral and communication indicators

Providers

Provider and model are selected from UI settings and applied to both single and batch pipelines.

Operational Notes

  • Batch runs update per-file progress in UI.
  • Threshold controls pass/fail highlighting.
  • Settings are locked until explicitly saved.
  • Results remain available in analytics history.
